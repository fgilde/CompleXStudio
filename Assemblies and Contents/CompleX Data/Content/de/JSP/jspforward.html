<jsp:forward>  

Die Aktion <jsp:forward> übergibt die Kontrolle über die Anforderungsverarbeitung an eine andere JSP-Seite oder ein anderes Servlet in derselben Webanwendung. Die Ausführung der aktuellen Seite wird beendet, so dass die Zielressource die vollständige Kontrolle über die Anforderung erhält.

Wenn die Aktion <jsp:forward> ausgeführt wird, wird jeglicher Antwortinhalt aus dem Puffer geleert. Wenn die Antwort bereits übernommen wurde (also teilweise an den Browser gesendet wurde), schlägt die Weiterleitung mit einer Ausnahme IllegalStateException fehl.

Mit dieser Aktion werden die Informationen zum URI-Pfad, die über das implizite Objekt request verfügbar sind, so angepasst, dass sie den Informationen zum URI-Pfad für die Zielressource entsprechen. Alle anderen Anforderungsinformationen werden unverändert beibehalten, so dass die Zielressource Zugriff auf alle ursprünglichen Parameter und Header hat, die mit der Anforderung übergeben wurden. Zusätzliche Parameter können über die Elemente <jsp:param> im Body des Elements <jsp:forward> an die Zielressource übergeben werden.

Beispiel:

<jsp:forward page="list.jsp" />
page String
Wert zur Anforderungszeit akzeptiert: ja

Obligatorisch. Ein seiten- oder kontextbezogener URI-Pfad, an den die Ressource die Benutzer weiterleitet.
