Beschreibung

Eine Instanz von ServletConfig wird von einem Webcontainer verwendet, um während der Initialisierung Informationen an ein Servlet oder eine JSP-Seite zu übermitteln. Die Konfigurationsinformationen enthalten Initialisierungsparameter (die im Bereitstellungsdeskriptor der Webanwendung definiert sind) und das Objekt ServletContext, das die Webanwendung darstellt, zu der das Servlet oder die JSP-Seite gehört.

getInitParameter()  
public String getInitParameter(String name)

Gibt einen String mit dem Wert des angegebenen Initialisierungsparameters für das Servlet oder die JSP-Seite zurück. Wenn der Parameter nicht vorhanden ist, wird null zurückgegeben.

getInitParameterNames()  
public java.util.Enumeration getInitParameterNames()

Gibt die Namen der Initialisierungsparameter für das Servlet oder die JSP-Seite als Enumeration von String-Objekten zurück. Wenn das Servlet keine Initialisierungsparameter enthält, wird eine leere Enumeration zurückgegeben.

getServletContext()  
public ServletContext getServletContext()

Gibt einen Verweis auf das Objekt ServletContext zurück, zu dem das Servlet oder die JSP-Seite gehört.

getServletName()  
public String getServletName()

Gibt den Namen dieser Servlet-Instanz oder JSP-Seite zurück. Der Name kann im Bereitstellungsdeskriptor der Webanwendung zugewiesen werden. Bei einer nicht registrierten (und deshalb unbenannten) Servlet-Instanz oder JSP-Seite wird der Name der Servlet-Klasse zurückgegeben.
