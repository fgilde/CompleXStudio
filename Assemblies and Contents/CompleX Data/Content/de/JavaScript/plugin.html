Ein plugin-Objekt stellt ein Plug-In dar, das beim Start beim Browser registriert wird. Im Allgemeinen wird über das navigator.plugins-Array auf einzelne Plug-Ins zugegriffen. Das navigator.mimeTypes-Array und die dazugehörigen Eigenschaften werden oft verwendet, um vor dem Laden von externen Inhalten festzustellen, ob der Browser über das gewünschte Plug-In verfügt. Die meisten Eigenschaften bieten über Skripts Zugriff auf Informationen, die sonst im Fenster Info über Plug-Ins angezeigt werden (dieses Fenster wird in Navigator über das Menü Hilfe, in IE/Macintosh über das Fenster File Helper Settings geöffnet). IE für Windows verwendet eine andere Technik und ermittelt anhand des für das Laden von ActiveX-Steuerelementen zuständigen Objektelements, ob externe Medien wiedergegeben werden können.

filenameNN 3 IE 5(Mac) DOM nicht zutreffend

Schreibgeschützt

Gibt den Dateinamen der Plug-In-Binärdatei zurück. In Win32-Versionen von Navigator wird der vollständige Pfadname, beim Macintosh (Navigator und IE) nur der Dateiname zurückgegeben.

 
Beispiel
  
var file = navigator.plugins[2].filename;
 
Wert

Zeichenfolge.
 
Standardwert

Keiner.

lengthNN 3 IE 5(Mac) DOM nicht zutreffend

Schreibgeschützt

Gibt die vom Plug-In unterstützte Anzahl von MIME-Typen zurück. Diese Eigenschaft ist nicht mit der Eigenschaft length des gesamten navigator.plugins-Arrays zu verwechseln, die angibt, wie viele plugin-Objekte der Browser erkennt.

 
Beispiel
  
var howManyMIMEs = navigator.plugins[2].length;
 
Wert

Eine ganze Zahl.
 
Standardwert

Keiner.

nameNN 3 IE 5(Mac) DOM nicht zutreffend

Schreibgeschützt

Gibt den Namen zurück, der dem Plug-In vom Hersteller zugewiesen wurde. Es ist jedoch nicht garantiert, dass ein für mehrere Betriebssysteme entworfenes Plug-In in allen Versionen denselben Namen hat.

 
Beispiel
  
var pName = navigator.plugins[2].name;
 
Wert

Eine ganze Zahl.
 
Standardwert

Keiner.
