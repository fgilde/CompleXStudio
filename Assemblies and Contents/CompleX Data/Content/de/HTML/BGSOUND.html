<BGSOUND>

HTML-End-Tag: Optional  

Mit diesem nur bei Internet Explorer vorhandenem Attribut wird eine Audiodatei definiert, die im Hintergrund abgespielt wird, während der Benutzer die Seite besucht. Das Element ist nur innerhalb des head-Elements zulässig. Hiermit werden die Eigenschaften zur Steuerung der Lautstärke und zur Wiederholung des Soundtracks festgelegt, nachdem die Audiodatei geladen wurde. Auch wenn ein End-Tag optional ist, besteht dafür kein Bedarf, weil alle Attribute für die Audiowiedergabe im Anfangs-Tag festgelegt sind. Das einzige Attribut, das mit anderen Elementen gemeinsam verwendet wird, ist id.

Wenn Sie dieses Tag verwenden, sollte der Benutzer ausdrücklich die Wahl haben, Hintergrundsounds zu deaktivieren, und die Vorgabe sollte "deaktiviert" sein. In Büroumgebungen kann es ungünstig sein (wenn nicht sogar peinlich), wenn ein Computer unerwartet Musik oder Klänge wiedergibt. Denken Sie auch daran, dass es vermutlich zu Verzögerungen beim Download kommt.

 
Beispiel
  
<BGSOUND src="tunes/mazeppa.mid">

[window.]document.getElementById(elementID)

balanceNN nicht zutreffend IE 4 HTML nicht zutreffend  

balance="signedInteger"

Optional  

Mit diesem Wert wird die Klangverteilung zwischen dem linken und rechten Lautsprecher festgelegt. Nachdem dieses Attribut im Element festgelegt wurde, kann der Wert nicht mehr durch ein Skript geändert werden.

 
Beispiel
  
<BGSOUND src="tunes/mazeppa.mid" balance="+2500">
 
Wert

Eine Integerwert mit Vorzeichen zwischen -10000 und +10000. Bei dem Wert 0 befindet sich die Balanceeinstellung in der Mitte zwischen der linken und rechten Seite. Ein negativer Wert verschiebt das Gewicht auf die linke Seite, ein positiver auf die rechte Seite.

 
Standardwert

0
 
Objektmodellreferenz
  
[window.] document.getElementsByTagName("bgsound")[0].balance

loopNN nicht zutreffend IE 3 HTML nicht zutreffend  

loop="integer"

Optional  

Legt fest, wie oft die Audiodatei wiedergegeben wird. Wenn das Attribut fehlt oder einen anderen Wert als -1 besitzt, wird der Klang mindestens einmal abgespielt. Beim Wert -1 wird die Audiodatei so lange wiederholt, bis die Seite geschlossen wird. Entgegen den Informationen im Microsoft Internet Explorer SDK scheint es keine Möglichkeit zu geben, die Audiodatei vorab zu laden, ohne sie wiederzugeben.

 
Beispiel
  
<BGSOUND src="tunes/mazeppa.mid" loop="3">
 
Wert

Zur einmaligen Wiedergabe kein Wert erforderlich. Auch beim Wert 0 wird die Datei nur einmal abgespielt. Bei Werten über Null wird die Datei entsprechend häufig wiedergegeben. Wenn Sie -1 zuweisen, wiederholt sich die Datei unbegrenzt.

 
Standardwert

-1
 
Objektmodellreferenz
  
[window.] document.getElementsByTagName("bgsound")[0].loop

src="URL"

Optional  

URL für die Audiodatei, die wiedergegeben werden soll. Welche Arten von Audiodateien verwendet werden können, hängt vom Browser ab. Gängige Audioformate, darunter auch MIDI, werden vom Internet Explorer ohne weitere Plug-Ins unterstützt.

 
Beispiel
  
<BGSOUND src="tunes/beethoven.mid">
 
Wert

Jede gültige URL, einschließlich vollständige und relative URLs. Die Datei muss als MIME-Typ vorliegen, der vom Internet Explorer oder von einem Plug-In unterstützt wird.

 
Standardwert

Keiner.
 
Objektmodellreferenz
  
[window.] document.getElementsByTagName("bgsound")[0].src
volumeNN nicht zutreffend IE 4 HTML nicht zutreffend  

volume="signedInteger"

Optional  

Dieser Integerwert definiert die Lautstärke des Hintergrundsounds relativ zur maximalen Lautstärke, die in den Benutzereinstellungen auf dem Clientcomputer festgelegt ist. Die maximale Lautstärke (Wert 0) ist nur so hoch wie in der Systemsteuerung unter Sound eingestellt. Dieses Attribut kann bis auf -10.000 eingestellt werden (obwohl die meisten Benutzer schon bei wesentlich höheren Werten nichts mehr hören).

 
Beispiel
  
<BGSOUND src="tunes/beethoven.mid" volume="-500">
 
Wert

Integer mit Vorzeichen zwischen -10.000 und 0.
 
Standardwert

0
 
Objektmodellreferenz
  
[window.] document.getElementsByTagName("bgsound")[0].volume
