<LI>...</LI>

HTML-End-Tag: Optional  

Das Element li ist ein einzelnes Listenelement, dass innerhalb eines ol- oder ul-Listencontainers verschachtelt ist. Der äußere Container bestimmt, ob dem li-Objekt eine Zahl oder ein Buchstabe vorangestellt wird (bestimmte Reihenfolge) oder ein Symbol (keine bestimmte Reihenfolge). Eine spezielle Kategorie von Stylesheet-Attributen ist für die Formatierung von Listen vorgesehen.

Wenn Sie eine Stylesheet-Regel auf ein li-Element anwenden, um die Farbe in Navigator 4 zu ändern, wird nur das vorangestellte Symbol farblich gekennzeichnet. Damit auch der Text farblich gekennzeichnet wird, müssen Sie das li-Element innerhalb eines span-Elements verschachteln und den Stil auf das span-Element anwenden. Dies funktioniert auch in allen anderen CSS-fähigen Browsern.

 
Beispiel
  
<UL>
    <LI>Larry</LI>
    <LI>Moe</LI>
    <LI>Curly</LI>
</UL>

typeNN alle IE alle HTML 3.2  

type="labelType"

Optional  

Das Attribut type bietet ein gewisses Maß an Flexibilität bei der Darstellung des vorangestellten Symbols oder der Folgenummer im Browser. Werte werden in zwei Gruppen unterteilt, wobei eine Gruppe für ol- und die andere für ul-Objekte verwendet wird. Bei einer ungeordneten Liste (ul) können Sie festlegen, ob das vorangestellte Symbol ein gefüllter Kreis, ein leerer Kreis oder ein Quadrat sein soll. Bei einer geordneten Liste (ol) können Sie zwischen Buchstaben (groß und klein), römischen Ziffern (groß und klein) und arabischen Ziffern wählen. In HTML 4 wird dem Stylesheet-Attribut list-style-type Vorrang vor dem Attribut type eingeräumt.

Beachten Sie, dass in Browsern der Version 4 das Attribut type für ein li-Element den Typ für nachfolgende li-Elemente in der Liste bestimmt, wenn es nicht durch eine type-Attributeinstellung in einem anderen li-Element außer Kraft gesetzt wird. In neueren Versionen wirkt sich das Attribut nur auf das aktuelle li-Element aus. Im Allgemeinen ist es am Besten, das Attribut type des ol- oder ul-Elements zu setzen und diese Einstellung für alle verschachtelten Elemente zu übernehmen.

 
Beispiel
  
<LI type="square">Chicken Curry</LI>
 
Wert

Wenn dieses Attribut von einem ul-Element umgeben ist, lauten die möglichen Werte disc | circle | square. Wenn es von einem ol-Element umgeben ist, lauten die möglichen Werte A | a | I | i | 1. Die Nummerierung erfolgt automatisch, wie in der folgenden Tabelle dargestellt.

TypBeispiel
AA, B, C, ...
aa, b, c, ...
II, II, III, ...
ii, ii, iii, ...
11, 2, 3, ...
 
Standardwert

1 und disc.
 
Objektmodellreferenz

valueNN alle IE alle HTML 3.2  

value="number"

Optional  

Das Attribut value ist nur dann relevant, wenn das li-Element in einem ol-Element verschachtelt ist. Die Nummer, die als Startpunkt für die Sequenz der Einträge in einer geordneten Liste verwendet wird, kann manuell festgelegt werden. Dies ist besonders dann nützlich, wenn ein ol-Element durch Fließtext, der nicht Teil der Liste ist, unterbrochen wird.

Der diesem Attribut zugeordnete Wert ist zwar eine Zahl, wirkt sich aber nicht auf die type-Einstellung aus. Wird value beispielsweise auf 3 gesetzt und ist für type der Wert A angegeben, bedeutet dies, dass die Sequenz ab diesem li-Element mit dem Buchstaben C beginnt.

 
Beispiel
  
<LI value="3">Insert Tab C into Slot M. Tighten with a wingnut.</LI>
 
Wert

Beliebiger positiver Integerwert.
 
Standardwert

1
