<ISINDEX>

HTML-End-Tag: Unzulässig  

Das Element isindex gibt es bereits seit den Anfängen von HTML. In HTML 4 wird jedoch der input-Elementtyp text bevorzugt. Das Tag des isindex-Elements gehört in das head-Element. In modernen Browsern wird es als einfaches Textfeld zwischen zwei hr-Elementen wiedergegeben. Wenn ein Benutzer Text in das Feld eingibt und die role="bold">Eingabe-/Return-Taste drückt, wird der Inhalt des Feldes URL-kodiert (mit +-Zeichen anstelle der Leerzeichen) und mit der URL des aktuellen Dokument an den Server übermittelt. Ein CGI-Programm auf dem Server muss diese URL verarbeiten und den entsprechenden HTML-Code zurückgeben können, damit er im aktuellen Fenster oder Frame angezeigt wird.

 
Beispiel
  
<HEAD>
<ISINDEX prompt="Enter a search string:">
</HEAD>

lang="languageCode"

Optional  

Die für die Attributwerte und Inhalte des Elements verwendete Sprache. Mithilfe dieser Informationen ist der Browser besser in der Lage, bestimmte Details wie Ligaturen (sofern von der betreffenden Schrift unterstützt oder in der geschriebenen Sprache erforderlich), Anführungszeichen und Worttrennungen korrekt darzustellen. Andere Anwendungen und Suchmaschinen können anhand dieser Informationen Wörterbücher für die Rechtschreibprüfung auswählen und Indizes erstellen.

 
Beispiel
  
<SPAN lang="de">Deutsche Bundesbahn</SPAN>
 
Wert

Sprachcode, bei dem nicht zwischen Groß- und Kleinschreibung unterschieden wird.
 
Standardwert

Vorgabe des Browsers.
