Audio-/Video-OBJECT mit äquivalentem CONTENT  
Section 508 1194.22(a); WAI / WCAG 1.0 checkpoint 1.1
 
 
Problembeschreibung

Die OBJECT-Elemente mit eingebetteten Audio- oder Videodateien in diesem Dokument sollten eine entsprechende äquivalente Beschreibung zwischen den Tags <OBJECT> und </OBJECT> besitzen.

 
 
Überprüfung

Fügen Sie eine aussagekräftige Beschreibung in den <OBJECT>-Block ein.

Prüfen Sie insbesondere den Inhalt zwischen den Tags <OBJECT> und </OBJECT>:

  • Handelt es sich beim Inhalt um eine Textbeschreibung, sollte diese eine Transkription der Audio-/Videodatei sein.
  • Ist der Inhalt ein Hyperlink, sollte er auf eine Transkription verweisen.
  • Wenn der Inhalt ein <OBJECT>-Tag ist, sollte es sich um eine Audiodatei oder eine Transkription handeln.
 
 
Problemerklärung

Sie können äquivalente Beschreibungen für Audiodaten in Form eines Textabschnitts auf der Seite bereitstellen, der auf eine Transkription oder Beschreibung der Audiodatei verweist. Der Hyperlink zur Transkription sollte sich an einer gut sichtbaren Stelle, z. B. oben auf der Seite, befinden. Wenn ein Skript eine Audiodatei automatisch lädt, dann sollte es auch automatisch eine visuelle Anzeige laden, aus der hervorgeht, dass die Datei gerade abgespielt wird, sowie eine Beschreibung oder Transkription der Audiodatei bereitstellen.

Das Design des OBJECT-Elements bietet einen Mechanismus zum Definieren von alternativen Objektdarstellungen. In jeder Deklaration eines eingebetteten OBJECT-Elements können alternative Inhaltstypen angegeben werden. Wenn ein Browser das äußerste OBJECT nicht darstellen kann, wird stattdessen die Darstellung der Inhalte versucht, bei denen es sich z. B. um ein weiteres OBJECT-Element handeln kann.
