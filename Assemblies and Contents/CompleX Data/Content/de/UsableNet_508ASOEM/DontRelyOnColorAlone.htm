Farbe ist nicht entscheidend.  
Section 508 1194.22(c); WAI / WCAG 1.0 checkpoint 2.1
 
 
Problembeschreibung

Die Seite enthält Farben. Stellen Sie sicher, dass die Vermittlung der Informationen auf der Seite nicht von den Farben abhängig ist.

 
 
Überprüfung

Achten Sie darauf, dass Benutzer die Seite selbst dann verstehen und bedienen können, wenn sie die Farben nicht wahrnehmen oder unterscheiden können.

Im Folgenden finden Sie einige einfache Schritte zum Testen Ihrer Seite:

  • Zeigen Sie die Seite auf einem Schwarzweißbildschirm an, und betrachten Sie dabei alle Seitenelemente.
  • Drucken Sie die Seite auf einem Schwarzweißdrucker.
  • Durchsuchen Sie die Seite, und entfernen Sie Anweisungen wie "klicken Sie auf die rote Schaltfläche".
 
 
Problemerklärung

Es gibt viele Situationen, in denen eine Person keine Farben sehen kann. Dazu gehören:

  • Schlechte Kombination von Vordergrund-/Hintergrundfarbe auf der Seite
  • Benutzer verwenden einen textbasierten Browser.
  • Benutzer verwenden einen Lesebrowser.
  • Benutzer verwenden eine Schwarzweißanzeige, wie einen PDA oder ein Mobiltelefon.
  • Farbenblinde Personen
  • Sehbehinderte oder blinde Personen

Farben sollten auf Seiten lediglich zu dekorativen Zwecken oder zum Verbessern von Grafiken eingesetzt werden. Sie dürfen keine Informationsträger sein.
