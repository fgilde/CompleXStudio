Informationen zum allgemeinen Layout einer Site bereitstellen  
WAI / WCAG 1.0 Priorität 2 Checkpunkt 13.3
 
 
Problembeschreibung

Sofern eine Site noch keine Sitemap hat, fügen Sie eine solche Sitemap hinzu und beschreiben Sie, wie die Inhalte in der Site angeordnet sind. Heben Sie die Zugänglichkeitsfunktionen hervor, die auf der Site zur Verfügung stehen.

 
 
Überprüfung

Fügen Sie also entweder eine grafische Sitemap ein, erstellen Sie eine verschachtelte Liste (wie das Inhaltsverzeichnis eines Buches) oder stellen Sie sie auf andere effiziente Weise aus Kategoriebeschreibungen von Informationsobjekten zusammen.

Sofern auf der Seite implementierte Zugänglichkeitsfunktionen beschrieben werden, sollten die Benutzer bestimmte assistive Technologien konfigurieren können, mit denen sie die Seiten anzeigen möchten.

 
 
Problemerklärung

Betrachten Sie die Sitemap als eine Art Inhaltsverzeichnis der Site. Für Besucher bietet sich damit eine hervorragende Möglichkeit, das Layout einer Site zu verstehen.

Bedenken Sie auch, dass Besucher mit Sehstörungen länger brauchen, um eine Site zu erfassen, da sie dem Inhalt der von ihnen besuchten Seiten erst zuhören müssen. Das dauert natürlich länger, als nur einen kurzen Blick darauf zu werfen.
