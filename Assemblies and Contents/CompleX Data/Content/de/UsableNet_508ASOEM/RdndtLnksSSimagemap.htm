Für serverbasierte Imagemaps werden Hyperlinks benötigt.  
Section 508 1194.22(e); WAI / WCAG 1.0 checkpoint 1.2
 
 
Problembeschreibung

Die Seite enthält eine serverbasierte Imagemap. Überprüfen Sie, ob jeder aktive Bereich der Imagemap über einen redundanten Textlink verfügt.

 
 
Überprüfung

Vergewissern Sie sich, dass jeder aktive Bereich der Imagemap als redundanter Hyperlink in der Seite doppelt vorhanden ist. Sie müssen die Implementierung der Imagemap auf dem Server prüfen, um herauszufinden, welche URLs aktiviert werden, wenn der Benutzer auf einen aktiven Bereich klickt.

 
 
Problemerklärung

Serverbasierte Imagemaps verursachen verschiedene Probleme in Bezug auf die Zugänglichkeit:

  • Zur Nutzung serverbasierter Imagemaps sind bestimmte Eingabegeräte, wie z. B. eine Maus, erforderlich, die bestimmten Benutzern in einem bestimmten Kontext möglicherweise nicht zur Verfügung stehen (z. B. Benutzern von Mobiltelefonen oder sprachgesteuerten Browsern).
  • Bestimmte Benutzer können in bestimmten Situationen möglicherweise nicht genau auf die gewünschte Stelle der Map klicken (z. B. Benutzer mit motorischen Einschränkungen, oder mobile (gehende, fahrende, stehende) Benutzer.
  • Die Hyperlinks serverbasierter Imagemaps sind auf dem Server verborgen und können weder von Browsern noch von Eingabehilfen bearbeitet werden. Dadurch kann der Browser dem Benutzer keine alternativen Hyperlinks anbieten.
  • Serverbasierte Imagemaps sind langsamer als clientbasierte Maps, da bei jeder Benutzereingabe eine zusätzliche Verbindung zum Server benötigt wird.

Aus diesen Gründen wird von der Verwendung serverbasierter Imagemaps stark abgeraten. Die einzige Ausnahme bilden aktive Bereiche, deren Formen so unregelmäßig sind, dass sie von clientbasierten Imagemaps nicht unterstützt werden.
