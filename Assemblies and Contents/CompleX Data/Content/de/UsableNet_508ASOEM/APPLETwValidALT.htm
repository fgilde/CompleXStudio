APPLET mit gültigem ALT-Attribut  
Section 508 1194.22(a); WAI / WCAG 1.0 checkpoint 1.1
 
 
Problembeschreibung

Applets in diesem Dokument haben keine entsprechenden gültigen Textbeschreibungen. Eine gültige Beschreibung ist eine Zeichenfolge im ALT-Attribut, bei der es sich nicht um folgende Zeichenfolgen handeln darf:

  • eine leere Zeichenfolge ("")
  • Eine Zeichenfolge mit einem oder mehreren Leerzeichen (" ")
  • Name der Datei, die das Applet enthält
  • die Dateigröße des Applets
 
 
Problemlösung

Fügen Sie das ALT-Attribut in das APPLET-Tag ein.

Die ALT-Beschreibung sollte dieselben Informationen wie das Applet selbst vermitteln und die Funktion des Applets auf der Seite erklären: Warum das Applet enthalten ist, welche Funktion übernommen wird und wie die Informationen ausgegeben werden.

 
 
Problemerklärung

Aufgrund folgender Ursachen kann ein Applet möglicherweise nicht ausgeführt werden:

  • Manche Benutzer zeigen die Seite auf PDAs oder Mobiltelefonen an, die Applets nicht ausführen können.
  • Manche Applets verwenden Bibliotheken, die für ein bestimmtes Betriebssystem geschrieben wurden, z. B. Windows, Mac OS oder Linux.
  • Manche Benutzer grafischer Browser deaktivieren Applets aus Sicherheitsgründen.

Mit dem ALT-Attribut können Sie den Benutzern das Verhalten und den Inhalt des Applets beschreiben.
