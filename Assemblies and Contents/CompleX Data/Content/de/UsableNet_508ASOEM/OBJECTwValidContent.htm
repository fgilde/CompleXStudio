OBJECT mit gültigem CONTENT  
Section 508 1194.22(a); WAI / WCAG 1.0 checkpoint 1.1
 
 
Problembeschreibung

Die Seite enthält OBJECT-Elemente ohne gültige Beschreibung zwischen den Tags <OBJECT> und </OBJECT>.

 
 
Problemlösung

Fügen Sie eine aussagekräftige Beschreibung in den <OBJECT>-Block ein.

Die ALT-Beschreibung sollte dieselben Informationen wie das eingebettete Objekt vermitteln und die Funktion des Objekts auf der Seite erklären: Warum das Objekt enthalten ist, welche Funktion übernommen wird und wie die Informationen ausgegeben werden.

 
 
Problemerklärung

Das Design des OBJECT-Elements bietet einen Mechanismus zum Definieren von alternativen Objektdarstellungen. In jeder Deklaration eines eingebetteten OBJECT-Elements können alternative Inhaltstypen angegeben werden. Wenn ein Browser das äußerste OBJECT nicht darstellen kann, wird stattdessen die Darstellung der Inhalte versucht, bei denen es sich z. B. um ein weiteres OBJECT-Element handeln kann.

Objekte in diesem Dokument haben keine entsprechenden gültigen Beschreibungen. Eine gültige Beschreibung ist ein HTML-Ausdruck zwischen den Tags <OBJECT> und </OBJECT> und beinhaltet Folgendes:

  • Mindestens ein Wort
  • Ein weiteres Objekt
  • Einen Hyperlink zu einer gültigen alternativen Darstellung

Eine gültige Beschreibung enthält keine Platzhalter (siehe http://www.w3.org/TR/AERT#AppendixC).
