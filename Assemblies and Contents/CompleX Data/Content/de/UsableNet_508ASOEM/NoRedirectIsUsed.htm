Keine automatische Weiterleitung  
Section 508 1194.22(p); WAI / WCAG 1.0 checkpoint 7.5
 
 
Problembeschreibung

Die Seite wird nach einer bestimmten Zeit automatisch aktualisiert, oder nach einer bestimmten Zeit wird automatisch eine neue Seite geladen. Sie sollten dieses Verhalten entfernen.

 
 
Problemlösung

Solange Browser dem Benutzer keine Möglichkeit zum Deaktivieren der automatischen Aktualisierung bieten, sollten Sie keine Funktionen zur automatischen Aktualisierung bzw. Weiterleitung verwenden. Wenn Sie eine automatische Weiterleitung benötigen, sollten Sie diese mit den vom Server bereitgestellten Funktionen umsetzen. Dadurch wird die Zugänglichkeit der Seiten nicht eingeschränkt, da die Seite nur bei Anforderung einer neuen Seite durch den Benutzer geändert würde. Wenn dies nicht möglich ist, sollten Sie dem Benutzer die Möglichkeit bieten, auch auf der automatisch geladenen Seite auf die Elemente der aktuellen Seite zuzugreifen. Dadurch kann der Benutzer bei einem Seitenwechsel weiterhin auf diese Elemente zugreifen.

 
 
Problemerklärung

Automatisch aktualisierte Seiten können behinderte Benutzer oder Benutzer, die auf Verfahren angewiesen sind, die normale Bedienungsmuster verhindern, vor große Probleme stellen. So funktionieren Bildschirmlesegeräte bei automatisch aktualisierten Seiten möglicherweise nicht korrekt. Benutzer mit physischen Einschränkungen können sich möglicherweise nicht schnell oder genau genug durch den Seiteninhalt oder die Navigationselemente bewegen. Darüber hinaus kann es zu Problemen kommen, wenn Benutzer nur langsam lesen können, eine sehr langsame Internetverbindung nutzen, oder wenn sie einen kleinen Bildschirm verwenden und dadurch langsamer lesen müssen. Solange Browser dem Benutzer keine Möglichkeit zum Deaktivieren der automatischen Aktualisierung bieten, sollten Sie die automatische Aktualisierung nicht verwenden.
